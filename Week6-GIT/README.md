# GIT — Version Control Hands-on (Mandatory)

Five command-line labs covering the mandatory GIT hands-on. Each `.md` is a walkthrough
with the exact Git Bash commands and the expected outcome.

| # | Lab | File |
|---|-----|------|
| 1 | Configuration & first commit | `01-git-config-and-first-commit.md` |
| 2 | Ignoring files with `.gitignore` | `02-gitignore.md` |
| 3 | Branching & merging | `03-branching-and-merging.md` |
| 4 | Merge-conflict resolution | `04-merge-conflict-resolution.md` |
| 5 | Clean up & push to remote | `05-cleanup-and-push-to-remote.md` |

> These are interactive Git labs (they involve a GitHub/GitLab account and, for 3–4, a visual
> merge tool such as P4Merge). The commands here are exactly what you run in Git Bash to
> complete each lab; run them in a scratch `GitDemo` repo to reproduce the results.
